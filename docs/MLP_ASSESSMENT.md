# DailyMemory - MLP (Minimum Lovable Product) 평가

> 평가일: 2026-04-07

## 현재 점수

| 요소 | 점수 | 현재 상태 |
|---|---|---|
| 핵심 가치 전달 | 6/10 | 기능은 많지만 "왜 써야 하는지"가 한눈에 안 옴 |
| 첫 경험 (FTUE) | 4/10 | 온보딩 후 빈 화면. 뭘 해야 할지 모름 |
| 핵심 루프 | 5/10 | 녹음→분석→저장은 되지만, 다시 돌아올 이유가 약함 |
| 감정적 연결 | 3/10 | 기능 중심이지 감성이 없음. 일기앱인데 차갑다 |
| 디자인/폴리시 | 4/10 | 기본 SwiftUI 컴포넌트, 브랜드 아이덴티티 없음 |
| 차별화 | 6/10 | AI 분석 + 시맨틱 검색은 독특하지만 체감이 약함 |

## 핵심 문제: "기능은 많은데, 사랑받을 이유가 없다"

현재는 MVP (Minimum Viable Product)에 가깝다. MLP가 되려면 아래 개선이 필요.

---

## 개선 항목

### 1. 빈 화면 문제 (Critical)
첫 로그인 후 Home이 텅 비어있음. 사용자는 3초 안에 떠남.

**필요:**
- 첫 기록을 유도하는 인터랙티브 가이드
- "오늘 하루 어땠어?" 같은 따뜻한 프롬프트
- 샘플 데이터로 앱의 가치를 미리 보여주기

### 2. 다시 돌아올 이유 (Critical)
현재 사용자가 앱을 다시 열 동기가 없음.

**필요:**
- 매일 저녁 알림: "오늘 하루를 기록해보세요" (설정에 토글은 있지만 실제 미구현)
- 아침 인사이트: Airflow DAG가 만든 리포트를 Home에 표시
- 스트릭: 연속 기록 일수 표시 (게이미피케이션)

### 3. 감성/브랜드 부재 (Important)
일기앱은 감성이 생명인데, UI가 너무 기능적.

**필요:**
- 커스텀 앱 아이콘 + 브랜드 컬러
- 타이포그래피 (기본 시스템 폰트 → 감성적 폰트)
- 기록 시 부드러운 애니메이션/햅틱
- "저장됨" 순간의 만족감 (confetti, 체크 애니메이션)

### 4. AI의 "와" 순간이 약함 (Important)
AI 분석은 기술적으로 잘 되지만, 사용자가 감동을 못 느낌.

**필요:**
- 분석 결과를 예쁜 카드로 보여주기 (현재는 텍스트 나열)
- "이번 달 Jake를 5번 만났네요!" 같은 인사이트를 Home에 자연스럽게 표시
- 검색 결과에 타임라인 시각화

### 5. 핵심 플로우가 너무 많은 단계 (Important)
기록하려면: + → 모드 선택 → 입력 → Analyze → 결과 확인 → Save (6단계)

**필요:**
- 원터치 녹음: + 버튼 길게 누르면 바로 녹음 시작
- 텍스트 입력 후 Save하면 자동으로 백그라운드 분석 (결과 확인 단계 스킵)

---

## MLP 달성 로드맵

| 순서 | 작업 | 효과 | 예상 소요 |
|---|---|---|---|
| 1 | Home 빈 화면 개선 + 첫 기록 유도 | 이탈 방지 | 0.5일 |
| 2 | 매일 저녁 기록 알림 (Daily Prompt) | 리텐션 | 0.5일 |
| 3 | 기록 플로우 단순화 (자동 백그라운드 분석) | UX | 1일 |
| 4 | 앱 아이콘 + 브랜드 디자인 | 첫인상 | 1일 |
| 5 | Home에 인사이트 카드 표시 (Airflow 리포트) | 재방문 동기 | 1일 |
| 6 | 저장 순간 마이크로 인터랙션 | 감성 | 0.5일 |

**1-3번만 해도 MLP에 근접. 1-6번 완료하면 MLP 달성.**
